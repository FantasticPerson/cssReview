浮动元素周围的外边距不会合并

```
如果要浮动一个非替换元素，则必须为该元素设置一个宽度
```

```
如果一个浮动元素与正常流中的内容发生重叠会怎么样呢？

行内元素与一个浮动框重叠时，其边框、背景和内容都在该浮动元素‘之上’显示

块框与一个浮动元素重叠时，其边框和背景在该浮动元素‘之下’，而内容在浮动元素之上显示
```

```
1.如果position是relative或static，包含块则由最近的块级框、表单元格或行内块祖先框的内容边界构成

2.对于一个非根元素，如果其position值是absolute，包含块设置为最近的position值不是static的祖先元素
```

```
使用top,right,bottom,left来描述四个边的放置位置，那么元素的宽度和高度将由这些偏移隐含确定。  
同时定义了这四个值之后  width height属性将对元素的布局没有任何作用，当然，倘若向元素增加了外边距，边框或内边距，此时，如果为height，width指定了显示的值，就会对布局产生作用。
```

```
内容裁剪 clip
可以设置负长度，这会使裁剪向上向左延伸了，不过延伸区域不会有任何内容

完全可以超越下边界和右边界 但是不能超越上边界和左边界
```

```
visibility
hidden 隐藏
如果祖先设置为hidden，但是其后代设置为 visible 还是可以正常显示的
```

```
元素绝对定位时，如果除bottom外某个任意偏移属性设置为auto，会放置在它的静态位置，比如 top设置为auto，那么顶端的纵坐标就是静态位置时的纵坐标
```

```
如果设置了margin值  比如说设置了 margin-left的值 那么 left的值就会被忽略，并重置right的值
```
```
relative 
如果一个相对定位元素过度受限
比如：
strong {position:relative;top:10px;bottom:20px;}
此时bottom是 -top 即 -10

right总是等于-left(从左往右读的语言中)
left总是等于-right(从右往左读的语言中)
```
